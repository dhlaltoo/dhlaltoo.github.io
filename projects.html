My Comic List

My Comic List allows users to manage lists of comic book series supplied by the Comic Vine API. The front end was designed using Bootstrap and is hosted on a Node.js Express server. Users are able to submit validated queries to the API via AJAX. All CRUD functionality, as well as user creation and authentication, was handled by a separate Express REST server, which queried a MySQL database.

This project was a huge undertaking, and served as an excellent opportunity to tie together many of the concepts and technologies I learned while in school. Node.js was independently researched and incorporated, which is something I am proud of.

Trip Tracker

Trip Tracker was written in vanilla JavaScript and was my first attempt at making API calls. The application makes a request to the Google Directions API based on validated user input: transit mode, origin, and destination. The JSON response is then parsed and the travel steps are iterated over and written out to the DOM. Font Awesome icons are appended according to transit mode.

Completion of this application cemented my interest in web development: it allowed me to use a powerful API to create a practical tool.

Humans Vs. Zombies!

This program was a simple modified 2D predator-prey simulation. Humans and zombies both extend the greater organism class, and are cast back and forth throughout the flow of the program. Each class follows specific sets of rules dictating how they move, reproduce, and expire, among other functions. The goal of the project was to have both populations survive for at least 100 turns: I never experienced an extinction event, so the counter was capped it at 150.

Humans Vs Zombies was one of the most rewarding assignments I completed during my school career. It was a challenging concept and very difficult to debug, but watching it in action was truly satisfying.

Rippopotamus

This project features a hungry hippopotamus that is controlled by key input. The object of the game is to eat 20 fruits before consuming 10 pieces of garbage. It is written in C# and uses the GDI+ API from Microsoft for graphics processing. The game also has sound effects and audio, including a looping 8-bit rendition of Toto's 'Africa' (for an enhanced user experience). Rippopotamus is another demonstration of object oriented principles and C# coding.

Asynchronous Chat

A simple TCP/IP chat application with a WPF interface. The application handles connections asynchronously using multithreading. Chat sessions are logged using a custom build library that implements a logging interface. Apache log4net is used for dependency injection in order to quickly swap out other logging libraries.

This project was a continuation of a similar synchronous client/server chat program. The progression illustrated how the use of delegates can be used to execute a method asynchronously.

Objectvania

Objectvania was my first foray into object oriented programming. This included concepts such as abstraction, encapsulation, and inheritance. It was written in Java with a Swing based GUI.

This set the stage for my interest in web development and creating tools that all sorts of people may use.
